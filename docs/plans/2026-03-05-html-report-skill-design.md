# HTML Report Skill Design

**Date:** 2026-03-05
**Skill:** `coding-session-reporter:html-report`

## Overview

A Claude skill that guides generating a self-contained HTML report from data returned by the coding-session-reporter MCP server. Triggered by natural language (e.g. "generate a report for this week"). Writes output to `/tmp` and opens it in the browser.

## Approach

Intent + tools reference (Option B): the skill maps user intent to MCP tool calls and defines what makes a good report, but lets Claude choose structure and visual style based on the data retrieved.

## Section 1: Intent → MCP Tool Mapping

| Intent | MCP tools |
|--------|-----------|
| "What did I work on today/this week?" | `get_sessions_summary` with date range |
| "Show me activity across projects" | `list_projects` + `get_sessions_summary` |
| "Deep dive on project X" | `get_session_logs` (with `userMessages`, `toolsUsed`) |
| "What subagents ran?" | `list_subagents` |
| "Find sessions about topic Y" | `search_sessions` |

## Section 2: Report Structure

Adapt sections to the data retrieved — include a section only if data warrants it:

- **Header** — date range, project filter, total sessions / hours
- **Timeline or table** — sessions grouped by day or project, with duration, first prompt, message count
- **Highlights** — most active day, longest session, most-used tools (when `toolsUsed` available)
- **Deep sections** — only when data warrants: subagent breakdown, key messages, search matches
- **Footer** — "Generated by MCP Coding Insights"

## Section 3: HTML Output Guidelines

- Generate **self-contained HTML** — inline all CSS
- Use CDN (e.g. Chart.js) only when data clearly benefits from a chart
- Write to `/tmp/session-report-<date>.html`, open with `open <file>` (macOS)
- Scale design to data volume: small standup → simple table; multi-project dashboard → cards/chart
- Use semantic HTML (`<table>`, `<section>`, `<time>`)
- Compute durations from `startedAt`/`endedAt` inline

## Full Flow

1. Parse user intent (date range, project filter, report type)
2. Call minimum MCP tools needed
3. Compute derived metrics (durations, totals)
4. Choose appropriate HTML structure for the data
5. Write to `/tmp`, open in browser
