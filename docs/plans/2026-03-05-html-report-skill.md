---

# HTML Report Generator

## Overview

Generate a self-contained HTML report from coding session data retrieved via the `coding-session-reporter` MCP server. Write the file to `/tmp` and open it in the browser.

## Flow

1. Parse intent → pick MCP tools
2. Call minimum tools needed
3. Compute derived metrics
4. Build HTML scaled to the data
5. Write to `/tmp/session-report-YYYY-MM-DD.html`
6. Run: `open /tmp/session-report-YYYY-MM-DD.html`

## Step 1: Map Intent to MCP Tools

| User says | MCP tools to call |
|-----------|------------------|
| "this week" / "today" / "last 7 days" | `get_sessions_summary` with `from`/`to` date range |
| "across all projects" / "overview" | `list_projects` then `get_sessions_summary` per project |
| "deep dive on [project]" / "full breakdown" | `get_session_logs` with `fields: ["sessionId","project","gitBranch","startedAt","endedAt","firstPrompt","userMessages","toolsUsed"]` |
| "subagents" / "agents that ran" | `list_subagents` |
| "find sessions about [topic]" | `search_sessions` with `query` |

**Date range calculation:**
- "today" → `from: start of today ISO`, `to: now ISO`
- "this week" → `from: last Monday ISO`, `to: now ISO`
- "last 7 days" → `from: 7 days ago ISO`, `to: now ISO`
- Always use ISO 8601 format: `2026-03-01T00:00:00.000Z`

## Step 2: Compute Derived Metrics

From the raw data, compute before building HTML:
- Session duration: `new Date(endedAt) - new Date(startedAt)` → format as `Xh Ym`
- Total hours per day: sum durations for sessions sharing a date
- Most active day: day with highest total duration
- Most-used tools: count occurrences across `toolsUsed` arrays

Skip metrics you don't have data for. Never invent placeholder values.

## Step 3: Build HTML

Scale the report to the data:

**Small (1-5 sessions, single day):** Simple table with session rows. No charts needed.

**Medium (1-2 weeks, 1-3 projects):** Sessions grouped by day. Highlight bar (most active day, total time). Optional tool breakdown list.

**Large (multiple projects, 2+ weeks):** Cards per project with session counts and hours. Consider a Chart.js bar chart for daily activity (load from CDN only if chart adds clear value).

### Required sections (include only if data exists):

```html
<!-- Header -->
<header>
  <h1>Coding Session Report</h1>
  <p class="meta">[date range] · [N sessions] · [X hours total]</p>
</header>

<!-- Timeline or table — always include -->
<section id="sessions">...</section>

<!-- Highlights — include if >3 sessions -->
<section id="highlights">
  Most active day: [day] ([Xh])
  Longest session: [firstPrompt truncated] ([Xh Ym])
  Most-used tools: [tool1, tool2, tool3]
</section>

<!-- Deep sections — include only if data warrants -->
<section id="subagents">...</section>
<section id="search-results">...</section>

<!-- Footer — always include, exact text -->
<footer>Generated by MCP Coding Insights</footer>
```

### HTML quality rules:
- Inline all CSS — no `<link>` tags, no external stylesheets
- Use semantic elements: `<table>`, `<section>`, `<time datetime="ISO">`, `<header>`, `<footer>`
- Use CDN only for Chart.js, only when a chart genuinely helps
- Truncate long `firstPrompt` strings to ~80 chars with ellipsis
- Format durations as `2h 14m`, not `134.3 minutes`

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Using today's date for "this week" instead of last Monday | Compute `from` as the most recent Monday at midnight |
| Including empty sections | Only render a section if you have data for it |
| Forgetting to open the file | Always end with `open /tmp/session-report-YYYY-MM-DD.html` |
| Footer text wrong | Must be exactly: `Generated by MCP Coding Insights` |
| Parallel sessions inflating hours | Note in report if sessions overlap (same project, same time) |
```
